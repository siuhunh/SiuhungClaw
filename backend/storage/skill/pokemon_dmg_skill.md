
# 宝可梦属性相克与招式威力（宝可梦冠军 / 主系列对战）

## 功能说明

帮助用户计算 **属性相克倍率**（含双属性乘算）、理解 **STAB（本系加成）** 与 **招式威力/分类** 等对战常识；需要招式名、宝可梦名或完整相克表细节时，用 **fetch_url** 拉取神奇宝贝百科（52poke）页面或 MediaWiki API，**不要编造**图鉴中不存在的数值。

## 核心规则（优先用逻辑回答）

### 1. 单属性相克倍率

对 **防守方某一属性**，查「**进攻招式属性** × **该防守属性**」的倍率，取值仅为 **0、½、1、2**（第四世代起无 4× 单格；双属性才出现 4× 等乘积）。

### 2. 双属性宝可梦（最常见问法）

最终倍率 = **对属性 A 的倍率 × 对属性 B 的倍率**（顺序无关）。

示例：火系招式打 **冰 + 草** → 对冰 2×、对草 2× → **2 × 2 = 4 倍**。

### 3. STAB（与属性表分开）

- 若 **招式属性** 与 **攻击方宝可梦** 的某一属性相同，通常威力阶段有 **本系加成**（主系列一般为 **×1.5**；具体世代/规则若用户说明则以该世代为准）。
- **属性相克表本身**只回答「招式属性 vs 防守方属性」；用户问「总伤害」时再单独乘上 STAB、道具、能力、场地等。

### 4. 何时不必联网

用户已给出 **进攻招式属性 + 防守方双属性** 且仅问相克倍率时，**直接按上式心算**即可，无需 `fetch_url`。

## 使用 fetch_url（神奇宝贝百科）

### 4.1 一般页面（HTML → 工具内转为可读文本）

- 词条 URL：`https://wiki.52poke.com/wiki/` + **URL 编码**后的标题（中文需编码）。
- 示例：`闪焰冲锋` → `https://wiki.52poke.com/wiki/%E9%97%AA%E7%84%B0%E5%86%B2%E9%94%8B`（招式名以百科实际标题为准，可能带「（招式）」后缀）。
- **说明**：`fetch_url` 对 HTML 会做清洗，返回的是**正文文本片段**，不是结构化 JSON；从中抓取「属性」「威力」「命中」「分类」等字段。

### 4.2 需要 JSON 时（可选）

MediaWiki API 示例（返回 JSON，便于解析）：

`https://wiki.52poke.com/api.php?action=query&prop=revisions&rvprop=content&rvslots=main&format=json&titles=` + 编码后的标题

仍应用 `fetch_url` 传入**完整 URL**；响应为 JSON 字符串时再解析 `query.pages` 下内容。

### 4.3 完整相克表

需要查全表或核对冷门组合时，可抓取百科中的属性相克相关页面（如「属性相克表」等标题），**以页面正文为准**。

## 回复格式建议

1. **只问倍率**：先给 **数字结论**，再一行写出 **对属性 A、对属性 B** 各为多少倍、如何相乘。
2. **问招式**：列出 **属性、威力、命中、分类**（物理/特殊/变化）及百科中注明的 **追加效果**（若有）。
3. **不确定世代或特殊规则**（如太晶、某些形态）：明确说 **「按默认主系列规则 / 需你补充世代或模式」**，避免武断。

## 示例

**倍率（不联网）**

```
用户：火属性招式打暴雪王（冰+草）属性相克多少倍？
助手：对冰 2×、对草 2×，合计 2×2 = 4 倍（仅属性相克，不含 STAB 等）。
```

**结合百科**

```
用户：闪焰冲锋威力多少？
助手：（fetch_url 拉取对应词条后）按当前百科数据：威力 xxx，属性 火，…（若拉取失败则说明失败原因，勿杜撰）。
```

## 注意

- 不同作品（《宝可梦冠军》与正作排位、宝可梦 GO 等）规则可能不同；用户指定游戏/世代时以该作为准。
- 百科词条标题与重定向以实际 URL 为准；搜索不到时可换用简称或加「（招式）」「（属性）」等后缀再编码尝试。
